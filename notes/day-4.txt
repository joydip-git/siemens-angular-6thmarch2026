service:
--------------------------------------
dependency injection of service

create a service class with some methods and common data
inform angular about the service class for DI
provider => which provides or creates instance of the service
	it needs to know about the service class
	it needs to know about a token for the service class
		- service class itself can act as a token
		- you can create a custom token for the service

	wherever in your app the token is used, there an instance of te service will supplied
interface IServiceContract
class SomeService:IServiceContract{
}


{
	provide: SomeService, <- token (service class itself is being used as token)
	useClass: SomeService <- service class that you are registering
}
@Component({....})
class SomeComponent{
	private ss:SomeService;
	constructor(ss:SomeService){
		this.ss = ss;
	}
	
}

{
	provide: 'SERVICE_TOKEN', <- token (string token)
	useClass: SomeService <- service class that you are registering
}

@Component({....})
class SomeComponent{
	
	private ss:IServiceContract;
	constructor(@Inject('SERVICE_TOKEN') ss:IServiceContract){
		this.ss = ss;
	}
	
}


any template/component lifecycle:
---------------------------------------
	mounting:
		1. constructor
		2. changes: gets fired when the component receives data from its parent (if any) through @Input() property, for the very first time (after constructor call) and every other time in the lifecycle of the component
		3. init: gets fired for the very first time, after the constructor call and mounting happened and after changes. it is not fired any more till the destruction.

	update:
		changes

	unmounting and destroy:
		destroy


X		OnX			ngOnX
-------------------------------------------------
init   		OnInit  		ngOnInit
changes    	OnChanges  		ngOnChanges
destroy		OnDestroy		ngOnDestroy


HTTP Request:
----------------------
package: @angular/common/http
HttpClient -> built-in service class
provideHttpClient() -> returns you the provider for HttpClient which you register in the providers array either at the component level  or at the root level
{provide:HttpClient,useClass:HttpClient }

HttpClientModule -> providers:[HttpClient] -> deprecated


methods of HttpClient:
----------------------------
get():Observable	-> sends an HTTP request with GET method
post	-> sends an HTTP request with POST method	
put	-> sends an HTTP request with PUT method
delete
patch
options
etc....


all methods of HttpClient class return "Observable" type object for response

when the request is over the "observable" gets the data or else an error is thrown if something goes wrong (such as wrong url etc.)

your service class must return the observable as returned by one of the HttpClient method

a component must subscribe to that observable as returned by the service class method by using "subscribe()" method of the "Observable"

a component must pass an object (with collection of callbacks for success, failure etc.) to the "subsribe()" method as an argument

this object is known as "Observer". one of the callbacks in "observer" gets executed at a time depending on the success or failure.

the "subscribe()"  method of the "Observable" object returns an instance of "Subscription" which you must release before the component gets destroyed by calling "unsubscribe()" method of the "Subscription" object
